Enhancement Two

The artifact being used is a project from earlier in my coursework, CS – 320 Software testing. The artifact handles the ability to add contacts, tasks, and appointments and store them to an ArrayList. Methods are present to allow for retrieving, deleting, and updating the fields of each of the three classes. These methods are in the model classes and service classes. As it was a software testing course, there are also testing classes for each of the model and service classes using JUnit testing.

This item was selected because I saw it as a surface level program which had a significant amount of room to demonstrate growth. The code needing refactoring throughout, a non-persistent ArrayList was used instead of a relational database, and the testing performed was limited. Ultimately it was a shell of a program that could be updated to serve a valuable purpose and demonstrate a wide variety of skills to put in my ePortfolio. Specifically in this milestone the the six test classes for the model and service classes were refactored to align with the updates performed last enhancement. Also, the testing themselves were improved to test each CRUD operation as well as validation of fields during object creation. The artifact ahs improved readability as well as maintainability.

I had originally planned to meet more course outcomes, specifically those regarding improved performance with regards to search methods, however I decided to integrate these into next weeks enhancement when the database is introduced. The ultimate goal remains the same, though the time of the integration is changed.

During this milestone I focused almost entirely on the testing, and I learned that it is much easier to integrate testing into the program as you are building it, instead of all at once in the middle or end of development. The writing and updating could be repetitive at times though this helped hammer home how important it is to be consistent across the multiple classes that are working in the same way. The repetition was a good lesson in how the testing needs to be sufficient across the board.
